# Data Profile

Use this profile when the repo primarily manages data pipelines, analytics workflows, notebooks, model training, ETL, or data products.

## Signals

- jobs, pipelines, notebooks, dbt, or dataset transforms dominate
- reproducibility and lineage matter more than a primary app runtime
- schemas, sources, and execution environments are central

## Default Planning Artifacts

- tooling survey
- data flow or module system doc
- milestone ledger
- task specs for risky pipeline or schema work

## Milestone Archetypes

- data source and workflow decisions
- first reproducible pipeline proof
- core transforms or models complete
- reliability and data-quality hardening
- follow-up optimization track

## Hardening Priorities

- source and sink contracts
- local bootstrap and reproducibility
- schema evolution
- fixtures or sample datasets
- validation and observability

## Verification Defaults

- lint or format
- schema validation
- pipeline smoke run
- representative dataset test
- reproducibility or determinism check

## Tooling-Survey Categories

- orchestration
- transformation framework
- storage and warehouse
- scheduler or queueing
- validation and lineage
- notebook or experiment workflow

## Task-Spec Threshold

Require a task spec for schema changes, pipeline topology changes, new sources or sinks, reproducibility-sensitive work, or jobs with high execution cost.
