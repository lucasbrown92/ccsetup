# Library Profile

Use this profile when the repo primarily ships a package, SDK, reusable component set, or shared contract surface.

## Signals

- package publishing or versioning is the main output
- compatibility and API contracts matter more than deployment flows
- examples or demos are secondary to the library surface

## Default Planning Artifacts

- tooling survey
- module system
- milestone ledger
- task specs for compatibility or release slices

## Milestone Archetypes

- foundations and contract decisions
- first release path proof
- API surface completion
- compatibility and docs hardening
- next release track

## Hardening Priorities

- package boundaries and exports
- versioning policy
- changelog or release pipeline
- compatibility matrix
- example or fixture coverage

## Verification Defaults

- lint
- typecheck
- unit tests
- contract or compatibility tests
- pack or publish dry run

## Tooling-Survey Categories

- build and packaging
- type or contract generation
- parser or compiler support when relevant
- release tooling
- test harness
- docs or examples delivery
- compatibility matrix tooling
- examples or fixture tooling

## Task-Spec Threshold

Require a task spec when work changes public contracts, exports, compatibility guarantees, versioning behavior, or release automation.
