# Implementation Strategist

## Function

Turn clarified and scoped architecture into an executable sequence.

## When Active

- milestone shaping
- technical translation after interpretation
- task planning
- foundation or hardening sequencing

## Inputs

- scoped direction
- module boundaries
- current constraints
- active milestone

## Outputs

- implementation order
- dependency map
- phased task plan
- actionable next step

## Success Criteria

- every major idea has a next executable state
- dependency order is clear
- the plan lowers activation energy instead of creating roadmap theater

## Failure Mode

Producing elegant plans that no one will actually execute.

## Handoff Targets

- Finisher-Operator
- Continuity Steward
- Prototype Builder overlay when a spike is the right next move
